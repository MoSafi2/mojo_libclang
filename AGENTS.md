# Repository Instructions

This repository generates raw Mojo FFI bindings for LLVM libclang with
`mojo-bindgen`.

## Command Wrapper

- Run shell commands through `rtk`.
- Prefer Pixi tasks over direct tool invocation.

## Main Commands

- Generate bindings: `rtk pixi run generate`
- Equivalent generate task: `rtk pixi run generate-libclang`
- Verify layout bindings: `rtk pixi run mojo build src/libclang_raw_layout_tests.mojo`
- A direct build of `src/libclang_raw.mojo` is expected to fail with
  `module does not contain a 'main' function`; this file is a library-style raw
  binding module.

## Generated Files

These files are generated by `scripts/generate_libclang_bindings.py`:

- `src/libclang_raw.mojo`
- `src/libclang_raw_layout_tests.mojo`
- `build/libclang_raw.ir.json`

Do not make persistent manual edits directly to generated binding files unless
the corresponding deterministic patch is also updated.

## Manual Patch Flow

The generator first runs `mojo-bindgen`, then applies patch files from
`patches/`.

Current patch:

- `patches/0001-libclang-raw-manual-abi.patch`

This patch contains the local libclang ABI fixes:

- By-value aggregate structs that need `RegisterPassable`.
- Flattened C pointer arrays into individual typed pointer fields.
- Scalar fields kept as normal scalar Mojo fields.
- `CXIndexOptions` kept opaque because it contains C bitfields.
- Callback ABI notes for callback typedefs and callback tables.
- Matching layout-test field offset updates.

If generated output changes and the patch no longer applies, `rtk pixi run
generate` should fail during `git apply --check`. Do not bypass that failure
unless intentionally inspecting pristine output.

Use `LIBCLANG_APPLY_PATCHES=0 rtk pixi run generate` only to inspect pristine
`mojo-bindgen` output. Do not commit pristine output as the final binding state
unless explicitly requested.

## ABI Rules For Manual Fixes

- If the C layout has scalar fields (`int`, `unsigned`, enums, floats, etc.),
  keep them as normal typed Mojo fields.
- If the C layout has pointers, keep them as typed Mojo pointer fields.
- If the C layout has an array of pointers in a by-value aggregate, flatten the
  array into numbered typed pointer fields instead of using `InlineArray`.
- Do not replace pointer fields with integer words unless explicitly requested
  and documented as an ABI workaround.
- Keep bitfield-containing structs opaque unless exact target ABI packing is
  represented and tested.

## Documentation

- `raw_bindings.md` is the project note for current generator limitations,
  downstream manual fixes, and upstream `mojo-bindgen` issues.
- Update `raw_bindings.md` whenever the generated ABI patch strategy changes.

## Git Hygiene

- The worktree may contain generated artifacts. Do not delete or revert user
  changes unless explicitly requested.
- When changing generated bindings, prefer updating the generator and patch
  files, then rerun `rtk pixi run generate`.
